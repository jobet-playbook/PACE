# Supabase Integration Setup

## Overview
The QA metrics system now uses Supabase for persistent storage instead of in-memory cache. Data sent from n8n is saved to the database and fetched dynamically by the dashboard.

## Database Schema
The `pace_qa_metrics` table structure:
```sql
create table public.pace_qa_metrics (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  output json null,
  last_30_business_days json null,
  rollback_windows json null,
  critical_wip_tickets json null,
  old_qa_wip_tickets json null,
  docs_id text null,
  constraint pace_qa_metrics_pkey primary key (id)
);
```

## Environment Variables Required

Add these to your `.env.local` file (already in your file):
```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

For Vercel deployment, add these as environment variables in your project settings.

## Changes Made

### 1. Added Supabase Client Library
- Added `@supabase/supabase-js` to `package.json`
- Created `lib/supabase.ts` for Supabase client configuration

### 2. Updated API Routes

#### `/api/qa-metrics` (POST)
- Now saves data to Supabase `pace_qa_metrics` table
- Returns database record ID and timestamp
- Handles all three payload formats from n8n

#### `/api/qa-metrics` (GET)
- Fetches the most recent record from Supabase
- Returns data with age information
- No longer uses in-memory cache

#### `/api/dashboard/qa-live` (GET)
- Fetches latest data from Supabase
- Transforms database records into dashboard format
- Provides real-time data from the database

### 3. Removed In-Memory Cache
- The `lib/qa-cache.ts` module is no longer needed
- All data is now persisted in Supabase

## Deployment Steps

1. **Install Dependencies**
   ```bash
   npm install
   ```

2. **Set Environment Variables in Vercel**
   - Go to your Vercel project settings
   - Add `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`

3. **Deploy**
   ```bash
   git add .
   git commit -m "Migrate to Supabase for persistent QA metrics storage"
   git push
   ```

## Benefits

✅ **Persistent Storage** - Data survives server restarts
✅ **Scalable** - No memory limitations
✅ **Historical Data** - All records are kept in the database
✅ **Real-time Updates** - Dashboard always shows latest data
✅ **No Cache Invalidation** - Data is always fresh from the database

## Testing

After deployment:
1. Send data from n8n workflow
2. Check Supabase dashboard to verify data is saved
3. Visit the QA tab on your dashboard
4. Verify live data is displayed correctly
